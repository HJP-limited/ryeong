# 🪪 명함 합성 데이터셋 생성기 (v2) 사양서

본 문서는 `generate.py` 스크립트를 통해 생성되는 명함 데이터셋의 기술적 사양과 데이터 구성 로직을 정리한 문서입니다.

---

## 1. 기본 생성 규격
* **해상도**: 300 DPI
* **물리적 크기 및 픽셀 해상도**:
    * **가로형 (Landscape)**: 90 × 50 mm ($1063 \times 591$ px)
    * **세로형 (Portrait)**: 50 × 90 mm ($591 \times 1063$ px)
* **출력 형식**: 300 DPI JPEG 이미지 및 xyxy 포맷 JSON 어노테이션

---

## 2. 디자인 구성 수치 (랜덤 요소)
명함 생성 시 아래 요소들이 무작위로 조합되어 데이터 다양성을 확보합니다.

| 구분 | 항목 | 수량 및 내용 |
| :--- | :--- | :--- |
| **레이아웃** | 전체 레이아웃 종류 | **12종** (가로형 6종 / 세로형 6종) |
| **폰트** | 사용 폰트 파일 | **13개** (Noto Sans 7종 / Noto Serif 6종) |
| **폰트 테마** | 조합 시너지 | **8종** (헤더와 바디 폰트의 굵기/스타일 조합) |
| **컬러** | 팔레트 조합 | **20종** (밝음, 어두움, 그라데이션 테마) |
| **배경** | 패턴 무늬 | **5종** (Plain, Dots, Stripes, Grid, Waves) |
| **로고** | 도형 스타일 | **6종** (원, 사각형, 다이아몬드, 육각형, 삼각형, 바) |

---

## 3. 콘텐츠 및 데이터 구성
### 🛠 콘텐츠 생성 전략
* **Faker (ko_KR, en_US)**: 성명, 주소, 상세 연락처, 이메일 ID 등 생성
* **Internal Lists**: 한국 기업 환경을 반영한 직급, 부서, 가상 회사명 사용

### 📊 데이터 리스트 규모
* **가상 회사명**: 한국어 30개 / 영어 25개
* **직급 (Title)**: 25개
* **부서 (Department)**: 15개
* **슬로건 (Slogan)**: 10개

### 📝 데이터 필드 및 등장 확률
데이터셋의 변별력을 위해 각 필드는 설정된 확률에 따라 포함 여부가 결정됩니다.
* **필수 항목 (100%)**: 성명, 회사명, 직위, 전화번호, 이메일
* **선택 항목**:
    * 주소 (65%), 휴대폰 (70%), 부서명 (55%), 웹사이트 (50%)
    * 팩스 (30%), QR코드 (25%), 슬로건 (20%), 로고 (80%)

---

## 4. 기술적 특징 및 저작권
* **어노테이션**: 모든 텍스트와 이미지 요소는 `xyxy` [xmin, ymin, xmax, ymax] 픽셀 좌표로 기록됩니다.
* **저작권 보호**:
    * **폰트**: OFL 1.1 라이선스의 Noto Sans/Serif CJK KR 폰트 사용
    * **로고/사명**: 실존 기업 정보를 배제하고 기하학적 도형 및 가상 리스트 활용
* **랜덤 로직**: **[팔레트 × 패턴 × 폰트테마 × 로고스타일 × 레이아웃]**의 조합을 통해 수만 가지 변수 생성

---

## 5. 라이브러리 스택
* **이미지 처리**: `Pillow (PIL)`
* **데이터 합성**: `Faker`
* **QR 코드**: `qrcode`
* **시스템**: `json`, `math`, `random`, `re`, `os`
